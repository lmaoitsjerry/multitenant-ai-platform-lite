## Milestone Summary

**Key Decisions:**
- D-18-01-01: Use asyncio.to_thread() for Supabase calls (Rationale: Simplest async wrapper)
- D-18-01-02: Use Optional[Any] for client return types (Rationale: Avoid importing external types)
- D-18-02-01: 60s TTL for CRM cache (Rationale: Balance freshness vs performance)
- D-18-02-02: Cache keys include tenant_id (Rationale: Tenant isolation)
- D-18-02-03: Graceful fallback when Redis unavailable (Rationale: Resilience)
- D-18-03-01: Pydantic models for response standardization (Rationale: FastAPI integration)

**Issues Resolved:**
- Race condition in DI caching (global dicts → lru_cache)
- Admin token timing attack (string comparison → hmac.compare_digest)
- N+1 queries in CRM search (loop → batch .in_())
- FAISS singleton thread safety (no lock → double-check locking)
- 15 bare exception handlers (replaced with proper logging)
- OpenAI outage crashes (no fallback → circuit breaker)
- Missing Supabase timeouts (none → 10s default)

**Issues Deferred:**
- PROD-21: Table name constants — deferred to v6 (current scale OK)
- PROD-23: MMR O(n²) optimization — deferred to v6 (current index size OK)

**Technical Debt Incurred:**
- response_models.py defined but not widely adopted by routes (cosmetic)
- PROD-22 was already addressed in Phase 16 with lru_cache maxsize=100
